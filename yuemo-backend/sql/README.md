# 月魔商城 - 数据库初始化工具

## 工具简介

这个工具用于快速初始化月魔商城项目的数据库，自动执行所有SQL文件创建所需的表结构。

## 环境要求

- Python 3.7 或更高版本
- pymysql 库

## 安装依赖

```bash
pip install -r requirements.txt
```

## 数据库配置

数据库连接信息在 `database_init.py` 中配置：

```python
db_config = {
    'host': '192.168.1.55',
    'port': 3306,
    'user': 'root',
    'password': 'jiangjing'
}
```

如果需要修改，请编辑 `database_init.py` 文件。

## 使用方法

### 方法一：直接运行（推荐）

```bash
python database_init.py
```

### 方法二：指定数据库地址

如果你的数据库配置不同，可以在运行时临时修改 `database_init.py` 中的配置。

## 功能特点

1. **自动识别SQL文件**：自动扫描当前目录下的所有 `.sql` 文件
2. **智能执行顺序**：按依赖顺序执行（init-all.sql 优先）
3. **错误处理**：
   - 跳过已存在的表和数据库（不会报错）
   - 显示详细的执行日志
   - 支持回滚机制
4. **安全性**：
   - 使用 `CREATE TABLE IF NOT EXISTS`
   - 使用 `CREATE DATABASE IF NOT EXISTS`
   - 可安全重复执行

## 执行顺序

工具会按以下顺序执行SQL文件：

1. `init-all.sql` - 全量表结构（开发环境用）
2. `yuemo_user.sql` - 用户模块数据库
3. `yuemo_product.sql` - 商品模块数据库
4. `yuemo_order.sql` - 订单模块数据库
5. `yuemo_payment.sql` - 支付模块数据库
6. `yuemo_cart.sql` - 购物车模块数据库
7. `yuemo_promotion.sql` - 促销模块数据库

## 常见问题

### Q: 连接失败怎么办？

1. 检查数据库服务是否启动
2. 确认IP地址和端口是否正确
3. 检查用户名和密码
4. 确认防火墙是否允许3306端口

### Q: 权限不足？

确保root用户有创建数据库和表的权限：
```sql
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%';
FLUSH PRIVILEGES;
```

### Q: 如何重新初始化？

由于使用了 `IF NOT EXISTS`，重复执行不会删除现有数据。如果需要完全重建，请先删除数据库：
```sql
DROP DATABASE IF EXISTS yuemo_mall;
```

然后重新运行工具。
