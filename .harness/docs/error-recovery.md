# 常见错误与恢复策略

## 后端错误

| 错误场景 | 恢复策略 |
|---|---|
| Maven 依赖下载失败 | 检查网络；检查 `settings.xml` 镜像配置；尝试 `-U` 强制更新 |
| Maven 编译错误 | 检查 Java 版本（17）；检查模块依赖顺序 |
| 单元测试失败 | 检查数据库连接配置；检查 MyBatis Mapper 路径 |

## 前端错误

| 错误场景 | 恢复策略 |
|---|---|
| npm install 失败 | 删除 `node_modules` 和 `package-lock.json`，重新 `npm install` |
| TypeScript 类型错误 | 检查 `tsconfig.json` 配置；检查依赖版本兼容性 |
| Vite 构建失败 | 检查 `vite.config.ts`；检查 import 路径 |

## Docker 错误

| 错误场景 | 恢复策略 |
|---|---|
| Docker build 失败 | 检查 Dockerfile 路径和 ARG；检查 COPY 源文件是否存在 |
| 镜像拉取失败 | 检查网络；检查镜像仓库地址 |
| 容器启动失败 | 检查端口占用；检查环境变量 |

## CI/CD 错误

| 错误场景 | 恢复策略 |
|---|---|
| CI 流水线卡住 | 检查 runner 是否在线；检查镜像是否能拉取；检查超时配置 |
| 部署后健康检查失败 | 检查容器日志 `docker compose logs`；检查端口映射；检查环境变量 |

## 验证失败处理流程

```
验证失败
  ↓
分析错误日志
  ↓
定位根因（编译错误？依赖缺失？配置错误？）
  ↓
修复代码
  ↓
重新验证（最多 3 次）
  ↓
3 次仍失败 → 暂停，向用户报告错误和已尝试的修复
```

## 状态恢复

- 每次修改文件后，记录修改清单（哪些文件、改了什么、为什么改）
- 如果任务中断，通过 `git diff` 恢复上下文
- 使用 `git status` 确认当前修改状态
